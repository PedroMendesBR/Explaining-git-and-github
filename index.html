Guia de Git e GitHub

Git

O Git é um sistema de controle de versões distribuído amplamente utilizado no desenvolvimento de software. Ele permite o gerenciamento eficiente de projetos, controlando alterações no código-fonte ao longo do tempo.

Algumas características importantes do Git incluem:

  • Distribuído: Cada desenvolvedor possui uma cópia completa do repositório, permitindo trabalho offline e descentralizado.
  • Rápido e Eficiente: Manipula grandes projetos com rapidez, usando mecanismos de branching e merging eficientes.
  • Flexível: Suporta diferentes fluxos de trabalho, desde simples a complexos.
  • Seguro: Toda alteração é verificada criptograficamente, garantindo integridade dos dados.

GitHub

O GitHub é uma plataforma de hospedagem de código-fonte e colaboração baseada em Git. Além de hospedar projetos Git, oferece funcionalidades adicionais como controle de acesso, colaboração social (através de pull requests e issues), integração contínua, entre outros.

Principais recursos do GitHub:

  • Repositórios: Hospedagem de projetos Git com controle de versão.
  • Colaboração: Facilita o trabalho em equipe através de pull requests, issues e comentários.
  • Integração Contínua: Integração com ferramentas de CI/CD para automação de builds e testes.
  • Comunidade: Grande comunidade de desenvolvedores compartilhando código e colaborando em projetos.

Comandos Git

Aqui estão os principais comandos Git que você precisará conhecer para usar efetivamente o Git no seu projeto:

  • git init: Inicializa um repositório Git local.
  • git clone: Clona um repositório Git existente para o seu computador.
  • git branch: Lista, cria ou deleta branches no repositório.
  • git checkout: Altera branches ou restaura arquivos do repositório.
  • git status: Mostra o estado atual do repositório.
  • git diff: Mostra as diferenças entre arquivos no diretório de trabalho e o que está no stage.
  • git add: Adiciona alterações do diretório de trabalho para o stage.
  • git commit: Registra as alterações no repositório.
  • git push: Envia as alterações locais para o repositório remoto.
  • git pull: Atualiza o repositório local com as alterações do repositório remoto.
  • git revert: Desfaz um commit específico, criando um novo commit.
  • git merge: Combina as alterações de uma branch com outra.
  • git stash: Armazena temporariamente alterações não finalizadas para limpar o diretório de trabalho.
  • git rm: Remove arquivos do diretório de trabalho e do stage.
  • git config: Configura opções do Git, como nome de usuário e email.
  • git reset: Desfaz alterações, revertendo o repositório para um estado anterior.
  • git remote: Gerencia repositórios remotos associados.
  • git fetch: Busca as atualizações do repositório remoto para o repositório local.
  • git show: Mostra informações sobre commits, tags ou objetos específicos.
  • git help: Mostra o manual de ajuda do Git.
  • git rebase: Reaplica commits em cima de outra base.
  • git blame: Mostra a autoria e a última modificação de cada linha de um arquivo.
  • git tag: Marca commits específicos com tags.

Configuração e Acesso ao GitHub

Para começar a utilizar o GitHub, siga estas etapas básicas de configuração:

  1. Crie uma conta no GitHub em github.com.
  2. Configure seu nome de usuário e email localmente:
    3. 
git config --global user.name "Seu Nome"
git config --global user.email "seu-email@example.com"
  3. Caso deseje, configure também outras opções como cores do terminal, ferramentas de merge padrão, etc.
  4. Crie um novo repositório no GitHub e siga as instruções para adicionar arquivos locais ao repositório remoto.
